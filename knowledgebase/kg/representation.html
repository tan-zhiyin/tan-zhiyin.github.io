  • The Rise of Ontologies:
    a need for a standardized way to define and categorize information.
    Ontological Concept Description
    Literal Represents concrete values like strings, numbers, etc.
    Class Represents a collection of individuals or entities.
    Property Relationships between classes or between classes and literals.
    Individual A specific instance of a class.
    Inheritance One class being a subclass of another.
    Transitivity If A relates to B and B relates to C, then A relates to C.
    Symmetry If A is related to B, then B is related to A.
    Inverse Properties Two properties that are the inverse of each other.
    Functional Properties Properties that can have only one value for each entity.
    Disjointness Two classes that have no members in common.
    Equivalence Two entities that are equivalent or identical.
    Cardinality Specifies the number of values a property can have.
    Domain & Range Specifies the class constraints for a property's subject and object.

    RDF, RDFS, and OWL

    Standards and languages that form the backbone of the semantic web. They expand on the ontology concept to allow for rich, structured, and linked data across the web.

    RDF is the base layer, representing data as triples. RDFS provides a basic typing system to this. OWL adds more complexity, allowing for detailed ontologies with constraints and relationships.

  • Semantic Web
    An extension of the current web, aiming to provide a standardized way to represent knowledge, making it understandable by machines.
    uses structures like RDF, OWL, etc., to allow machines to understand and reason with web content.

Ontologies

A formal naming and definition of types, properties, and interrelationships of the entities that exist for a particular domain.
